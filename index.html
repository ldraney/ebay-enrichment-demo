Phase 1-2: Data Ingestion

Daily full refresh from Powerlink SQL to Google Sheets, then IPN persistence and category resolution into Airtable Master Parts Table.

Phase 3: Physical Data

ShipStation API integration for weight and dimensions. Backfill missing data, then physically lock fields to prevent overwrites.

Phase 4: Rule Engine (Core)

8 rule types govern Item Specifics automation. AI classification with 75% confidence threshold. Escalation to ClickUp for manual review.

Phase 5-7: Pre-Publish

Pre-publish governance with approval gate, fitment extraction from HTML, and automated title/description generation. Nothing goes live without explicit approval.

Authority Boundaries

Airtable Master Parts Table

Source of truth for IPN-level data. Physical field locking.

eBay Listings (API) Table

Staging only. Temporary. Never source of truth.

ClickUp

Escalation path for low-confidence AI or manual-only fields.

The Rule Engine

Phase 4 is the hardest part. 8 rule types govern how Item Specifics get populated, with strict enforcement of confidence thresholds and field locking.

Rule Name Behavior Destination Lock?
F Fixed Known value for prefix. Write immediately. Airtable Yes
FsV Fixed Source → Variable Pull from Master Parts Table. If missing, write "Does Not Apply" to listing only. Listing No
VF Variable → Fixed AI determines value. If ≥75% confidence, write + lock. Skip dot-number IPNs. Airtable Yes
V1 Variable (Listing-Only) AI sources from inventory/title/condition. Listing only. Never lock. Listing No
V2 Variable (IPN Interpretation) AI interprets IPN via external lookup. Listing only. Listing No
VB Variable / Blank Optional field. AI fills if confident, else leave blank. Never lock. Listing No
VMF Variable → Manual → Fixed AI tries first. If <75%, create ClickUp task. Lock after manual entry. Airtable After Manual
MF Manual → Fixed Always human research. Create ClickUp task immediately. Lock after entry. Airtable After Manual

Critical Constraints

  • ✗ Never overwrite Fixed (F) values
  • ✗ Never write "Does Not Apply" to Master Parts Table
  • ✗ Excluded prefixes (900, 950, 999) never ingested
  • ✗ No silent updates, no reprocessing

Key Behaviors

  • ✓ 75% confidence threshold for all AI decisions
  • ✓ Physical locking = Airtable field protection
  • ✓ Dot-number IPNs (.xxxx) skip VF rules
  • ✓ ClickUp escalation for low confidence / manual fields

Implementation Approach

Here's how I'd tackle this project.

Phase 1: Foundation (Weeks 1-2)

  • • Set up Airtable schema with proper field types and views
  • • Build Powerlink → Sheets daily sync (Apps Script)
  • • Implement Sheets → Airtable IPN persistence logic
  • • Design Part Type Logic tables structure

Phase 2: Rule Engine Core

  • • Implement all 8 rule types as modular functions
  • • Build AI classification layer with confidence scoring
  • • Create physical field locking mechanism
  • • Integrate ClickUp API for task escalation

Phase 3: Publishing Pipeline

  • • Build pre-publish governance workflow
  • • Implement fitment extraction (HTML parsing)
  • • Create title/description generation templates
  • • Integrate eBay API for publishing

Key Architectural Decisions

  • 1. Event-driven processing — Each IPN change triggers rule evaluation, not batch processing
  • 2. Immutable audit log — Every field change is logged with timestamp, source, and confidence
  • 3. Fail-safe defaults — If in doubt, escalate to ClickUp rather than write incorrect data
  • 4. Modular rule engine — Each rule type is a pure function, easy to test and extend
